# D2R Multiplay 匠心准则 🛠️

> 我们在打磨一个可以传世的小工具。代码要干净，系统要轻松。

---

## 🚀 性能就是尊重

- **复用，别重造**：禁止在循环里写 `System::new_all()`。必须用 Tauri State 缓存系统实例。
- **精细化更新**：进程刷新只用 `ProcessRefreshKind::nothing()`。别让 CPU 觉得我们在无端骚扰它。

## 🏛️ 结构即正义

- **拒绝过度包装**：少于 20 行的代码别开新文件，直接合并。薄包装层是代码的脂肪。
- **井水不犯河水**：
    - `win32_safe/`：纯净的 Win32 API 封装。
    - `account.rs`：高层业务编排。
    - 底层不碰配置，高层不碰底层 API。

## 📝 记录心跳 (Logging)

- **别用 `println!`**：那是写作业。用 `tracing`（debug/info/warn/error），这是给生命留下的日志。

## 🛡️ 错误处理

- **有话直说**：报错要吐出人话。目前用字符串转换，未来统一收拢。

## 🎨 前端优雅

- **拒绝白屏**：窗口默认隐藏，挂载后再显示。
- **懒加载**：大组件用 `React.lazy()`。

## 🤝 协作契约 (Git)

- **有头有脸**：提交必须带前缀（`feat`, `fix`, `perf`, `docs`）。
- **人机一心**：Mimi（AI）在推送前必须听到“实施”指令。
